Color Usage Philosophy
Oro (Gold) — Use Sparingly

Reserve gold exclusively for: primary CTAs on dark backgrounds, premium badges (Luxury / Exclusive), decorative flourishes, and active selection states. Max 3 gold elements per viewport. Never use gold as a background color for large areas.

Travertine — The Dominant Surface

Travertine (#F5F0E8) is the primary background for 90% of the platform. It communicates warmth, stone, heritage. White is reserved for elevated card surfaces. Never use pure #FFFFFF as a page background — it undermines the editorial luxury feel.

Conversion-Guided Color

Primary CTAs use Obsidian (dark) on light pages and Oro on dark surfaces. The contrast hierarchy guides the eye: Gold → Obsidian → Terracotta → neutral. Limit CTA colors to a single accent per screen section.

Multilingual Clarity

Avoid conveying meaning through color alone. Always pair semantic colors with icons and text labels — essential for RTL languages and colorblind accessibility. Warning states use amber + icon; never amber alone.

Section 05

Design Principles

Three foundational principles that govern every design decision at TRAVEL-LAND.IT — from a single button to a full-page experience.

Principle 01
Elegance Through Simplicity
Every element earns its place. Luxury is expressed through restraint — generous white space, editorial typography, and carefully chosen imagery communicate more than decoration ever could.
✓ Do This
A single destination card with ample breathing room, a Cormorant headline, and one clear CTA. Silence is part of the design.
✕ Not This
A card crowded with icons, badges, multiple CTAs, and competing colors. Visual noise destroys the luxury signal.
Principle 02
Experience Before Transaction
The traveler must feel the destination before they're asked to book it. Immersive imagery, evocative copy, and unhurried pacing build the emotional foundation that makes conversion effortless.
✓ Do This
Hero section: full-bleed atmospheric image → evocative headline → story → then the CTA. Let them want it first.
✕ Not This
Opening with a price, a "Book Now" button, or a search form before any emotional context has been established.
Principle 03
Confidence Builds Conversion
Luxury customers don't respond to urgency — they respond to certainty. Every interaction must reinforce trust: clear information hierarchy, reassuring feedback, and frictionless flows.
✓ Do This
Success states are warm and specific: "Your journey is confirmed — ref #TL-8847. Maria will contact you within 24 hours."
✕ Not This
Generic "⚡ Only 1 left! Book in the next 10 minutes!" pressure tactics that undermine the premium positioning.
Section 05.2

Do's & Don'ts

✓ Do
Use Travertine as your page backgroundThe warm off-white communicates Italian stone heritage and creates editorial calm. It makes imagery and text breathe.
✕ Don't
Use pure white (#FFFFFF) as page backgroundClinical white strips the warmth and luxury character. It makes the platform feel like a generic booking engine rather than a curated service.
✓ Do
Lead with Cormorant Garamond for display textThe serif display font signals culture, heritage, and editorial sophistication — qualities the luxury traveler expects and trusts.
✕ Don't
Use Cormorant at small body sizesCormorant Garamond is exclusively for display (22px+). At 14px it becomes illegible. Use DM Sans for all body copy, UI labels, and functional text.
✓ Do
Use gold (Oro) for exactly one CTA per sectionScarcity of gold increases its perceived value. Reserve it for the highest-conversion moment in each screen region.
✕ Don't
Use gold as a background or decorationGold borders, gold gradients, and decorative gold motifs cheapen the brand and reduce the impact of the gold CTA.
✓ Do
Use --space-16 (64px) minimum between major sectionsGenerous vertical rhythm communicates premium quality. Let the content breathe.
✕ Don't
Compress sections to fit more content above the foldPrioritizing "information density" over elegance destroys the luxury experience. Let the scroll be part of the journey.
✓ Do
Always pair color-coded alerts with text + iconNever communicate warnings or errors through color alone — critical for accessibility (WCAG 1.4.1) and international readability.
✕ Don't
Use red borders as the only error indicator8% of men are colorblind. An invisible form error is a lost booking. Always use icons, text labels, and color together.
Section 05.3

Developer Implementation Guide

Naming Conventions

CSS tokens: kebab-case with semantic intent: --color-[name], --space-[n]

Component classes: BEM-inspired: .destination-card__title, .btn--gold

JS state: data attributes for JS hooks: data-state="open", never class-only state changes

Dark Mode Strategy

Use CSS custom property overrides within @media (prefers-color-scheme: dark) and also [data-theme="dark"] for manual toggle support.

All color tokens reference semantic names, never hardcoded hex values in components. Swap the token values, not the components.

Accessibility Checklist
✓ All interactive elements have visible focus rings (2px oro outline)
✓ Minimum touch target: 44×44px (Apple HIG standard)
✓ Color contrast: 4.5:1 AA minimum for all body text
✓ All images have descriptive alt attributes
✓ Semantic HTML: <nav>, <main>, <article>, landmark roles
✓ Form fields: labels programmatically linked via for/id
✓ Error messages: aria-live="polite" regions
✓ Accordion: aria-expanded, aria-controls pairs
✓ Modals: focus trap, Escape key dismissal
Performance & Image Strategy
✓ Hero images: AVIF → WebP → JPEG fallback chain
✓ Lazy load: all below-fold images with native loading="lazy"
✓ Art direction: <picture> with mobile-optimized crops
✓ LCP target: hero image ≤ 2.5s on 4G mobile
✓ Fonts: font-display: swap, preload critical weights only
✓ Subset fonts to Latin + Latin-Extended for multilingual
✓ Motion: respect prefers-reduced-motion — disable all transitions
Multilingual Considerations
Text Expansion German and Finnish can expand English by 35%. Use flexible containers (avoid fixed widths on text), and test all UI labels with German copy. Buttons must wrap gracefully.
RTL Support (Arabic) Use logical CSS properties: margin-inline-start not margin-left. Set dir="rtl" on <html> and use CSS :dir(rtl) selectors.
Font Fallbacks Cormorant Garamond has no Arabic or CJK glyphs. Define language-specific font stacks: Arabic → Noto Sans Arabic; Japanese → Noto Serif JP.
